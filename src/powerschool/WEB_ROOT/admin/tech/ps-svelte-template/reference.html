PowerSchool Svelte Template Reference

This reference guide provides documentation for using the Svelte Template plugin to create modern web components and applications within PowerSchool.

What is the Svelte Template?

The PowerSchool Svelte Template is a modern framework for building PowerSchool plugins using Svelte 5, TypeScript, and Vite. It allows you to create both full applications and individual reusable web components that can be embedded in PowerSchool pages.

Key Features

Svelte 5 with Runes

Uses the latest Svelte 5 features for reactive state management

TypeScript

Full TypeScript support for type safety and better developer experience

Standard CSS

Clean, vanilla CSS with no external dependencies for simple and maintainable styling

Shadow DOM Encapsulation

Components are isolated from PowerSchool's styles

Automatic Component Registration

Components in src/lib are automatically available as web components

PowerSchool Plugin Ready

Includes plugin.xml and proper output structure

Using Components in PowerSchool

Full Application

To use the full Svelte application in a PowerSchool page:

<!-- Include the script --> <script type="module" src="/your-plugin-name/index.js"></script> <!-- Replace 'your-plugin-name' with your package.json name --> <!-- Use the custom element --> <ps-svelte-app></ps-svelte-app>

Individual Components

To use individual components from the library:

<!-- Include the script --> <script type="module" src="/your-plugin-name/index.js"></script> <!-- Replace 'your-plugin-name' with your package.json name --> <!-- Use a component --> <svelte-counter></svelte-counter> <!-- With attributes/props --> <svelte-counter initial-count="10"></svelte-counter>

Creating New Components

To add a new component to the library:

  1. Create a new .svelte file in the src/lib directory
  2. Use Svelte 5's runes syntax for props and state
  3. That's it! The component will be automatically registered as a web component

Example Component Code

<script lang="ts"> // Define props using Svelte 5 runes const props = $props<{name?: string}>(); // Default value if name is not provided const displayName = props.name ?? 'World'; </script> <div class="greeting"> <h2>Hello, {displayName}!</h2> </div> <style> .greeting { padding: 1rem; border: 1px solid #ccc; border-radius: 0.5rem; background-color: #f9f9f9; } h2 { margin: 0; color: #333; } </style>

This component will be automatically available as <svelte-greeting> in PowerSchool:

<svelte-greeting name="PowerSchool"></svelte-greeting>

Creating Custom Pages

To create a custom page in PowerSchool using this template:

1. Add a page definition to your plugin.xml:

<page name="my-custom-page" content="my-custom-page.html" />

Advanced Usage

Component Attributes and Properties

HTML attributes are automatically converted to camelCase for Svelte props:

<!-- HTML attribute: data-value --> <svelte-component data-value="123"></svelte-component>

In your Svelte component:

<script lang="ts"> const props = $props<{dataValue?: string}>(); console.log(props.dataValue); // "123" </script>

Event Handling

To listen to events from your components in PowerSchool:

<svelte-counter id="myCounter"></svelte-counter> <script> document.getElementById('myCounter').addEventListener('count-changed', (event) => { console.log('New count:', event.detail.count); }); </script>

Troubleshooting

Component Not Showing Up

  • Make sure the script is properly loaded
  • Check browser console for errors
  • Verify that the component name matches the file name (with svelte- prefix)

Styling Issues

  • Remember that Shadow DOM isolates styles
  • Global styles from PowerSchool won't affect your components
  • Use the browser inspector to debug Shadow DOM styles
